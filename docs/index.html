Akmega ⚡

From C Firmware to GDSII Silicon in a Single Session.

The Thesis

The ATmega architecture is an industry titan, powering millions of devices worldwide. Yet, as my friend Mike Matera observed, it remains surprisingly opaque. Documentation is fragmented, toolchain visibility is low, and bootstrapping a custom core from scratch is often considered a high-friction endeavor.

Akmega was born from a simple question: Can we use AI as a force multiplier to bridge this documentation gap and ship a verified core in a matter of hours?

Insights from the Arena

In a previous attempt to build a larger RISC-V core, I hit a wall at physical placement. The complexity of modern ISA implementation often makes automated P&R (Place and Route) fail for solo developers without enterprise-grade tools.

Akmega represents a successful course correction. By focusing on a lean, 100% compatible 8-bit core and integrating modern AXI4-Lite buses, I achieved a clean physical layout on the Skywater 130nm process. It's a proof of concept for a new operating model: highly technical founders shipping verified silicon using AI to handle the cognitive load of architectural mapping.

The Specification

Compatibility

Verified against avr-gcc generated binaries. Supports word arithmetic and hardware multiplication.

Buses

Dual AXI4-Lite master interfaces for instruction fetch and data memory. SoC-ready by design.

Process

Synthesized and routed for Sky130 using OpenLane. Zero DRC/LVS violations in GDSII.

Verification

Bit-perfect matching between RTL simulation and a Python Functional Golden Model.
